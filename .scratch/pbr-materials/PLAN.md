# Plan: PBR materials (metallic-roughness) with texture maps

Status: **future / ready-for-agent** — scoped and mapped, not started. Design
decisions locked with the user; pipeline insertion points mapped against the
codebase. Pick this up with `/tdd` when ready. No code written yet.

## Decisions locked with the user

1. **Workflow: metallic-roughness** (the glTF/modern standard), not
   specular-glossiness. Reuses the existing GGX + Fresnel directly; one material
   spans plastic ↔ metal.
2. **First cut = the core four maps:** base color, normal, roughness, metallic.
   (AO, emissive, height/parallax are explicitly deferred — a later unit.)
3. **Asset binding = loose image files** referenced per-material in the scene
   `MaterialSpec` (fits the existing `TextureSpec::Image` + OBJ loader). **No glTF
   importer** — that's a separate feature in its own right.

## What already exists (reuse, don't rebuild)

- **GGX/VNDF microfacet** — `src/material/microfacet.rs`: `ggx_reflect(ray_dir, n,
  alpha, rng) -> (wo, cos_half, g2_over_g1)`, Smith masking, `MIRROR_ALPHA`.
- **Multi-lobe template** — `src/material/glossy.rs`: stochastic Fresnel lobe
  selection (coat-reflection vs diffuse base), reports the *sampled lobe*'s
  specular flag so NEE lights the diffuse part, `scattering_pdf` = bare cosine.
  **This is the exact shape the PBR material generalizes.**
- **Image textures + UV mapping** — `ImageTexture` (`src/texture/image_texture.rs`),
  `MappedTexture` tiling (`src/texture/mapped_texture.rs`), `Texture::value(u,v,p)
  -> Color`.
- **Triangles already carry** optional per-vertex normals + UVs
  (`Triangle::from_points_smooth_uv`); `Triangle::build` already computes the edge
  vectors `u = p2-p1`, `v = p3-p1` needed for tangents.
- **Integrator contract** (`src/integrator/mis.rs`): `scatter_lobe -> (scattered,
  atten, specular)`; specular lobe → BSDF-sampled, diffuse lobe → NEE-sampled with
  `scattering_pdf`. The diffuse *bounce direction* is generated by the integrator
  via `cosine_direction_from_uv(&hit.normal, …)` — see the normal-map subtlety.

## The BRDF (metallic-roughness), generalizing `Glossy`

At a hit, sample the material params (from textures at `(u,v)`): `base_color`
(linear RGB), `metallic ∈ [0,1]`, `roughness ∈ [0,1]`, and the (possibly
normal-mapped) shading normal.

- `F0 = lerp(vec3(0.04), base_color, metallic)` — dielectric reflectance 0.04 vs
  colored metal reflectance = base color.
- `diffuse = base_color · (1 − metallic)` — metals have no diffuse lobe.
- Two lobes, stochastically selected (as `Glossy` does): **specular** = GGX
  reflection weighted by *colored* Fresnel(F0) · `g2_over_g1`, reported specular;
  **diffuse** = Lambertian with `diffuse`, reported non-specular; `scattering_pdf`
  = the bare cosine pdf on the diffuse branch.

## Two load-bearing subtleties (flagged for the grill)

1. **Normal-map altitude — apply at HitRecord construction, not inside the
   material's scatter.** If the material perturbs the normal only inside
   `scatter_lobe`, the integrator's own diffuse bounce
   (`cosine_direction_from_uv(&hit.normal, …)`) and `scattering_pdf` still see the
   *geometric* normal → the sampled direction and its pdf disagree → **biased**.
   Fix: add a `Material::shading_normal(geo, tangent) -> Vec3` hook (default =
   geometric normal, so everything else stays bit-identical), which
   `World::intersect` calls so `HitRecord.normal` is already the perturbed shading
   normal — consistent across the integrator, both lobes, and the pdf.
   *(This is the deep-module decision; the grill should stress it.)*
2. **BRDF unbiasedness with a colored F0.** `Glossy`'s white-coat trick (select
   the lobe with its Fresnel probability, return the plain lobe albedo, and the
   Fresnel factor cancels) does **not** transfer unchanged to a *colored* F0 — the
   per-lobe selection probability and the returned weight must be worked out so the
   metal case stays unbiased. Prove it the same way shadow-rays Stage 2 did: a
   **tight MIS-vs-Naive gate** (the independent Naive oracle), including a metallic
   scene, before trusting any render.

## Pipeline insertion points (mapped against the code)

- **(a) Linear vs sRGB texture load.** sRGB decode is *unconditional* today —
  `src/texture/image_loader.rs:5-20` (`rgb8_to_linear`). Base color is sRGB
  (correct as-is); **normal / roughness / metallic maps must load raw-linear**.
  Thread a `ColorSpace` flag: field on `Asset` or `TextureSpec::Image`
  (`src/scene/texture.rs` — **append**, postcard keys by declaration order) →
  `build_image` (the single chokepoint, `src/scene/texture.rs:123-128`) →
  `ImageTexture::from_bytes` → the loader. Default sRGB keeps existing scenes
  bit-identical.
- **(b) Scalar sampling.** `Texture::value` returns `Color` only — no scalar
  accessor. Roughness/metallic pull one channel (e.g. `.x`) from the returned
  linear `Color`; decide a convention (red channel) in the material.
- **(c) New material spec.** **Append** `MaterialSpec::MetallicRoughness {
  base_color: TextureSpec, metallic: TextureSpec, roughness: TextureSpec, normal:
  Option<TextureSpec>, … }` after `DiffuseLight` (`src/scene/material.rs:13-33`);
  add its arm to `build()` (`:36-54`). New runtime material under `src/material/`.
- **(d) Tangents.** Add `tangent: Vec3` to `GeoHit` and `HitRecord`
  (`src/ray/hit_record.rs`; thread `GeoHit::new`, `HitRecord::new`, `from_geo`).
  Compute per-triangle in `Triangle::build` (`src/geometry/triangle.rs:63-91`) from
  edges `u`/`v` + UV deltas (`dp/du, dp/dv`); write onto `GeoHit` in `intersect`.
  Analytic tangents: Quad `self.u.unit()` (`src/geometry/quad.rs:79`), Sphere the
  longitude tangent (`src/geometry/sphere.rs:101-102`). Fallback when UVs are
  absent or the UV Jacobian is degenerate (mirror the degenerate-normal guard).

## Suggested slice plan (`/tdd`, dependency order)

1. **Tangent infrastructure** — `tangent` on GeoHit/HitRecord; per-triangle from
   UV gradients; analytic for quad/sphere; no-UV fallback. Bit-identical (nothing
   reads it yet). Test: tangent ⊥ normal, aligned with `dp/du`, unit-length.
2. **Metallic-roughness material, constant params** — the BRDF core (F0 lerp,
   diffuse suppression, GGX specular lobe + Lambertian diffuse lobe,
   `scatter_lobe`/`scattering_pdf`). Test the limits: `metallic=0` ≈ dielectric,
   `metallic=1` = colored metal with no diffuse; **MIS-vs-Naive unbiasedness gate
   incl. a metallic scene** (the correctness proof).
3. **`Material::shading_normal` hook + normal mapping** — default geometric normal
   (bit-identical); PBR material decodes the (linear) normal map through the
   tangent basis. Test: identity map `(0.5,0.5,1)` → unperturbed; a tilted map
   perturbs as expected; perturbed normal kept on the geometric side (no leaks).
4. **Linear texture loading** — `ColorSpace` flag through the image path; default
   sRGB bit-identical. Test: a linear-loaded image returns raw values.
5. **Texture-driven params + scene wiring** — sample base_color/metallic/roughness
   maps at the hit; `MaterialSpec::MetallicRoughness` variant + `build()`;
   end-to-end render of a PBR scene. Existing render pin unchanged (Cornell box has
   no PBR material); optionally add a new PBR characterization pin.
6. **`/simplify`** — reuse/altitude sweep, as with shadow-rays.

## Out of scope (later units)

- glTF importer (mesh + material + maps together).
- Specular-glossiness workflow.
- Anisotropy, clearcoat, sheen, transmission (glTF extensions).
- Mip-mapping / texture filtering beyond nearest (aliasing on minified maps).
