Reference

Env-map sampling glossary

The canonical words for this course. Lessons adhere to these.

The image

Texel
One pixel of the HDR environment map — a linear-light radiance for one patch of sky. “Texture element.” In code: an entry of EnvMap.data.
Equirectangular
The map’s layout: horizontal axis = longitude (compass), vertical axis = latitude (up/down). Stretches the poles.
Row / Column
A row = all texels at one latitude (a ring of sky at fixed height). A column = one longitude. Row 0 (top) = straight up (+Y) in this tracer.
Latitude θ (theta) / Longitude φ (phi)
θ = angle from straight up (0 = up, π = down). φ = compass angle around the vertical. A direction ⇄ (θ, φ) ⇄ (row, column).
Luminance
A single “brightness” number distilled from a texel’s RGB (a weighted sum). What we weight sampling by.
HDR
High dynamic range: texel values aren’t capped at 1.0, so a sun can be thousands× brighter than sky — the reason naive sampling is noisy.

Sampling by brightness

Importance sampling
Drawing samples where they matter (bright directions) instead of uniformly, then correcting for the bias with a pdf. Cuts variance.
CDF — cumulative distribution function
The running total of a pdf, climbing 0 → 1. Its cut-points carve [0, 1) into segments whose widths are the probabilities. Steep = high density.
Inverse transform sampling (inversion method)
Turn a uniform draw u into a sample by x = F⁻¹(u) — find which CDF segment u falls in. The pdf shapes the segments, not the draw. (Lesson 2.)
Distribution1D
PBRT's 1D sampler: a cumulative array + binary search + a pdf report. The building block Distribution2D stacks (one marginal + N conditionals).
Marginal distribution
The “which row” pick: probability of each row ∝ its total (weighted) brightness.
Conditional distribution
The “which column, given the row” pick: probability of each texel in the chosen row ∝ its brightness.
Distribution2D / PiecewiseConstant2D
PBRT’s structure holding the marginal + per-row conditionals, precomputed once so 2D weighted sampling is two cheap 1D lookups.
PDF (probability density)
How likely the sampler was to pick a given direction, per unit solid angle. Needed to un-bias importance sampling and to weight MIS.
Solid angle
The “area on the sphere of directions” a patch of sky covers (units: steradians). pdfs for lighting live in this measure.
Jacobian
The local area stretch factor of a mapping: output area ÷ input area near a point. For image→sphere it is 2π²·sin θ. (Lesson 3.)
Change of variables (for densities)
Remapping a density through a mapping: pout = pin / |Jacobian|. Density divides by the stretch factor — stretch lowers it, squish raises it.
sin θ (why it appears twice)
The stretch factor between image area and solid angle is 2π²·sin θ. It enters as a weight (× sin θ) when building the distribution, and in the pdf conversion image→direction (÷ sin θ). The two cancel, so you sample directions ∝ brightness. (Lessons 3–4.)

Using it in the integrator

NEE — next-event estimation
Sampling a direction toward a light (here, a bright sky direction) and shadow-testing it, rather than waiting for a bounce to find it.
MIS — multiple importance sampling
Combining two strategies (light-sampling + BSDF-sampling) so each covers the other’s weak cases, weighted so the result stays unbiased.
Power heuristic
The specific MIS weight this tracer uses (β = 2): a² / (a² + b²) over the two pdfs. Code: integrator::common::power_heuristic.
Firefly
A single blown-out bright pixel from a rare high-value sample — the symptom env-map importance sampling removes.
